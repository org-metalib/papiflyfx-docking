## Compatibility Policy

Breaking API/provider/session changes are acceptable when they improve architecture and runtime correctness.

## Objective
Implement **Phase 4 — persistence extension generalization** now that Phases 1-3 are complete and validated on-host. Replace the hardcoded ribbon session field with a namespaced extension model and tighten serializer shape validation so future dock extensions can persist state without adding bespoke fields to `DockSessionData`.

## Mandatory Scope (Phase 4)

1. **Phase 3 preflight validation (first)**
   - Re-run and record targeted validation before the schema refactor:
     - `./mvnw -pl papiflyfx-docking-api,papiflyfx-docking-docks -am compile`
     - `./mvnw -pl papiflyfx-docking-api,papiflyfx-docking-docks -am -Dtestfx.headless=true test`
     - `./mvnw -pl papiflyfx-docking-samples,papiflyfx-docking-github,papiflyfx-docking-hugo -am compile`
   - If preflight fails, fix regressions first and record outcomes in `progress.md` before new persistence work.

2. **Replace the hardcoded ribbon session field with a namespaced extension model**
   - Refactor `DockSessionData` so extension state is stored in a generic namespaced container rather than a dedicated `ribbon` property.
   - The serialized shape should make ownership explicit (for example `extensions.ribbon`, or an equivalent deterministic structure) and allow multiple extension payloads without adding new top-level fields.
   - Keep extension ordering deterministic.
   - Compatibility with the old top-level `ribbon` payload is optional and not required. If you skip a legacy bridge, document the intentional schema cutover in code/spec notes.

3. **Generalize the contributor SPI around extension keys/codecs**
   - Refactor `DockSessionStateContributor` away from the current “mutate the whole `DockSessionData` and call `withRibbon(...)`” pattern.
   - Introduce a stable namespace/key contract for contributors and a small codec/adapter surface so typed payloads (such as `RibbonSessionData`) encode/decode through the extension model instead of being hardwired into `DockSessionSerializer`.
   - Duplicate contributor namespaces must be explicit and deterministic: prefer fail-fast or clearly logged rejection over silent last-write-wins.
   - Preserve the current capture/restore lifecycle in `DefaultDockSessionService`.

4. **Move ribbon persistence onto the generalized extension path**
   - Update `RibbonSessionStateContributor` to persist ribbon state under a stable namespace (likely `ribbon`) via the new extension model.
   - `RibbonSessionData` may remain a typed DTO, but it should be encoded/decoded through the contributor/codec path rather than as a dedicated `DockSessionData` field.
   - Keep the Phase 2/3 ribbon invariants intact, especially ID-first QAT restore and graceful behavior when selected tabs or commands are temporarily unavailable.

5. **Add serializer shape-validation helpers**
   - Add reusable helpers in `DockSessionSerializer` (or a tightly scoped internal helper) for safe typed reads: map/list/string/boolean/number extraction, required vs optional fields, and clearer `IllegalArgumentException` messages when the root session shape is invalid.
   - Replace raw unchecked casts for floating/minimized/maximized/restore-hint and extension payload parsing with those helpers.
   - A malformed extension payload must not corrupt unrelated session state. Either drop that extension with a warning or fail its decode in a narrowly scoped way; do not let one bad extension poison layout restore.

6. **Phase 4 test closure**
   - Add/extend tests in `papiflyfx-docking-docks` for:
     - serializer round-trip with namespaced extensions and more than one extension entry
     - ribbon persistence round-trip through the new extension shape
     - duplicate extension namespace handling
     - malformed extension payload tolerance / validation coverage
     - malformed core session field diagnostics from the new shape helpers
   - Update existing ribbon/session persistence tests to assert the new JSON shape.
   - Keep tests deterministic under `-Dtestfx.headless=true`.
