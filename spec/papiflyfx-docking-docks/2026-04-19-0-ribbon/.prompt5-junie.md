## Objective
Implement Phase 5 of the Ribbon Toolbar plan: Persist ribbon state (minimized state, selected tab, QAT), update documentation, and perform final cross-module validation.

## Key Tasks

1.  **Ribbon State Persistence (`papiflyfx-docking-docks`):**
    *   Implement serialization for ribbon state:
        *   `minimized` (boolean).
        *   `selectedTabId` (String).
        *   `quickAccessCommandIds` (List<String>).
    *   Decide whether to extend existing docking session JSON or use a parallel payload (refer to `progress.md` handoff notes).
    *   Ensure backward compatibility: the framework must handle layouts without ribbon state and remain tolerant of missing providers or commands on restore.

2.  **Documentation Update:**
    *   Update `spec/papiflyfx-docking-docks/2026-04-19-0-ribbon/README.md` with a comprehensive overview of the Ribbon implementation.
    *   Add documentation for the Ribbon SPI (`RibbonProvider`, `PapiflyCommand`, etc.) in `papiflyfx-docking-api`.
    *   Update root `README.md` or module-level READMEs if necessary to mention the new Ribbon capability.

3.  **Regression Coverage & Validation:**
    *   Add tests for ribbon state persistence and restoration.
    *   Verify that missing command IDs in QAT or missing tabs on restore are handled gracefully.
    *   Add tests for resize/scaling transitions to ensure no regressions were introduced.
    *   Perform final manual GUI validation in the `papiflyfx-docking-samples` app:
        *   Verify adaptive/collapsed ribbon ergonomics.
        *   Verify contextual-tab transitions.
        *   Verify ribbon state persists across application restarts.

4.  **Final Polish:**
    *   Review `ribbon.css` for any final visual refinements.
    *   Update `spec/papiflyfx-docking-docks/2026-04-19-0-ribbon/progress.md` and `validation.md` (if it exists) to reflect 100% completion.

## Validation
*   Build all modules: `./mvnw clean package`.
*   Run all ribbon-related tests: `./mvnw -pl papiflyfx-docking-api,papiflyfx-docking-docks,papiflyfx-docking-github,papiflyfx-docking-hugo,papiflyfx-docking-samples -am test`.
*   Manual check: Run the sample app and verify that the ribbon state (e.g., minimized state, selected tab) is preserved after closing and reopening the app.

## Invariants
*   Session compatibility: Existing layouts must not break.
*   Version tolerance: Ribbon state restoration should not fail if a previously used module is no longer present.
*   UI consistency: Follow the repository's UI standards and token vocabulary.
