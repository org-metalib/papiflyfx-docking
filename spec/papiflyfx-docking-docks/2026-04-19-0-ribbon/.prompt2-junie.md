# Ribbon Implementation Phase 2: Ribbon Shell and Shared Visuals

Follow the design specified in `spec/papiflyfx-docking-docks/2026-04-19-0-ribbon/research-gemini.md` and `plan.md` to implement the Ribbon shell.

## Lead Agent
`@core-architect`

## Objective
Implement Phase 2 of the Ribbon Toolbar plan: Create the ribbon container, tab strip, group chrome, and Quick Access Toolbar in `papiflyfx-docking-docks`.

## Key Tasks

1.  **Ribbon Container & Components (`papiflyfx-docking-docks`):**
    *   Implement `Ribbon`: The top-level container (likely extending `Control`).
    *   Implement `RibbonTabStrip`: To manage and display `RibbonTab`s.
    *   Implement `RibbonGroup`: Container for controls within a tab, including the group label and optional dialog box launcher.
    *   Implement `QuickAccessToolbar` (QAT): Persistent toolbar for global actions.

2.  **Runtime Discovery & Wiring:**
    *   Implement `RibbonManager` (or similar) to handle `ServiceLoader` discovery of `RibbonProvider`s.
    *   Provide mechanism to populate the `Ribbon` with tabs and groups defined by the discovered providers.
    *   Initialize `RibbonContext` and pass it to providers during tab construction.

3.  **Visuals & Styling:**
    *   Create `ribbon.css` in `papiflyfx-docking-docks`.
    *   Use established `-pf-ui-*` CSS tokens for background, borders, and text colors.
    *   Define ribbon-specific CSS variables (e.g., `-fx-ribbon-accent`, `-fx-ribbon-background`) as proposed in the research doc.
    *   Ensure the ribbon is theme-aware (bind to `Theme` properties where appropriate).

4.  **Basic Layout:**
    *   Ensure tabs and groups are laid out horizontally.
    *   Implement "Minimized" mode where only tabs are visible.
    *   Note: Advanced adaptive scaling (LARGE/MEDIUM/SMALL/COLLAPSED) is for Phase 3, but the shell should be built to accommodate it.

5.  **Host Integration:**
    *   Provide a way to attach the `Ribbon` to a `DockManager` or a top-level application layout.

## Validation
*   Ensure the project builds: `./mvnw clean compile`.
*   Manual check: Add the `Ribbon` to a sample in `papiflyfx-docking-samples` to verify rendering.
*   Update `spec/papiflyfx-docking-docks/2026-04-19-0-ribbon/progress.md` after completion.

## Invariants
*   Visual consistency with `-pf-ui-*` standards.
*   Decoupling of command metadata (API) from visual nodes (Docks).
*   No breaking changes to existing docking layouts.
