# Ribbon Implementation Phase 3: Adaptive Layout and Collapsed Groups

Follow the design specified in `spec/papiflyfx-docking-docks/2026-04-19-0-ribbon/research-gemini.md` and `plan.md` to implement adaptive sizing for the Ribbon.

## Lead Agent
`@core-architect`

## Objective
Implement Phase 3 of the Ribbon Toolbar plan: Add adaptive scaling logic to `RibbonGroup` and support for collapsed group popups in `papiflyfx-docking-docks`.

## Key Tasks

1.  **Group Size Modes (`papiflyfx-docking-docks`):**
    *   Implement size modes: `LARGE`, `MEDIUM`, `SMALL`, and `COLLAPSED` for `RibbonGroup`.
    *   Define layout templates for each mode:
        *   `LARGE`: 32x32 icons + full labels.
        *   `MEDIUM`: 16x16 icons + full labels (stacked or side-by-side).
        *   `SMALL`: 16x16 icons only (tooltips for labels).
        *   `COLLAPSED`: Single 32x32 icon button representing the group.

2.  **Adaptive Scaling Engine:**
    *   Implement the reduction algorithm in `RibbonTabStrip` or `Ribbon`.
    *   Use `reductionPriority` from `RibbonGroupSpec` to decide which groups shrink first when width is constrained.
    *   Ensure smooth transitions between size modes during window resizing.

3.  **Collapsed Group Popups:**
    *   Implement the popup content for `COLLAPSED` groups.
    *   The popup should display all commands from the group in their `LARGE` or `MEDIUM` format to ensure full access.
    *   Ensure the popup is theme-aware and follows the visual style of the ribbon.

4.  **Visual Refinement:**
    *   Update `ribbon.css` to handle different size modes and their transitions.
    *   Ensure consistent spacing and alignment across all scaling states.

5.  **Testing & Validation:**
    *   Add TestFX/Monocle tests for resize transitions (e.g., verifying groups shrink in priority order).
    *   Add tests for collapsed group popup appearance and interaction.
    *   Update `spec/papiflyfx-docking-docks/2026-04-19-0-ribbon/progress.md` after completion.

## Validation
*   Build the project: `./mvnw clean compile`.
*   Run tests: `./mvnw -pl papiflyfx-docking-docks -am test`.
*   Manual check: Resize the sample app window to verify predictable group reduction and popup behavior.

## Invariants
*   No command should be unreachable regardless of ribbon width.
*   Preserve the SPI-first architecture (no runtime leaking into API).
*   Visual consistency with `-pf-ui-*` standards.
