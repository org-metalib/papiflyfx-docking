## Problem

In the `SamplesApp` demo application, the `Docks -> Ribbon Shell` sample shows clipped text in the ribbon:

- command labels such as `Paste`, `Copy`, `Duplicate`, and `Pin Preview` lose their lower glyph area
- group captions such as `Clipboard` are also clipped

Treat this as a ribbon runtime geometry defect in `papiflyfx-docking-docks`, not as a sample-data problem to hide by shortening labels.

## Primary goal

Make ribbon labels and group captions render cleanly at the default SamplesApp size and across the existing adaptive ribbon size modes.

## Constraints

1. Preserve the current ribbon SPI and provider contracts.
2. Keep the fix inside the owning runtime module when possible.
3. Do not ‚Äúfix‚Äù the issue by renaming or shortening sample labels.
4. Preserve command accessibility across `LARGE`, `MEDIUM`, `SMALL`, and `COLLAPSED` states.
5. Keep group launcher behavior and collapsed-popup behavior intact.
6. Continue using the existing `-pf-ui-*` CSS token vocabulary and current visual language.

## Expected approach

### 1. Reproduce and measure

- Reproduce the clipping in the SamplesApp ribbon shell if the environment allows it.
- Inspect the effective layout budget for:
  - large-mode controls
  - medium-mode controls
  - group footer/caption row
- Confirm whether the defect is caused by fixed control height, footer-row compression, baseline alignment, or a combination.

### 2. Fix control geometry

- Adjust large-mode geometry so stacked icon-plus-label controls have enough vertical room for real JavaFX font metrics.
- Revisit medium-mode presentation and choose one of these only if justified by the code and resulting behavior:
  - keep stacked labels and increase the height budget
  - switch to a denser inline icon-plus-label layout
- Prefer the smallest change that removes clipping without destabilizing adaptive layout behavior.

### 3. Fix group footer geometry

- Give the footer caption row an explicit vertical budget if needed.
- Ensure captions keep full descenders visible with and without a launcher button.
- Verify that footer alignment does not regress when launcher visibility changes.

### 4. Add regression coverage

- Add or extend a ribbon Fx test that exercises the same kind of labels visible in the sample fixture.
- Assert containment/visibility invariants after CSS and layout are applied.
- Avoid brittle pixel-perfect screenshot assertions unless they are clearly necessary.
- Re-run adaptive ribbon tests to confirm no regressions in collapse ordering or cached control reuse.
