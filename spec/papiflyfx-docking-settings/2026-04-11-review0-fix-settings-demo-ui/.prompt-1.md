# Agent 1 — `@ui-ux-designer`: Inject Theme Tokens and Standardize Settings UI

**Module scope:** `papiflyfx-docking-settings` only (read `papiflyfx-docking-api` for reference, do not modify it)

## Context

`settings.css` references `-pf-ui-*` CSS variables (`-pf-ui-surface-panel`, `-pf-ui-text-primary`, etc.)
but those variables are never defined on any ancestor node. Every rule resolves to nothing — invisible text,
missing backgrounds, broken controls.

The proven pattern for injecting these tokens exists in `GitHubToolbar.java:522-529`:

```java
private static String buildRootStyle(Theme baseTheme, GitHubToolbarTheme toolbarTheme) {
    return GitHubThemeSupport.themeVariables(toolbarTheme)
        + UiStyleSupport.metricVariables()
        + UiStyleSupport.fontVariables(baseTheme.contentFont())
        + String.format(Locale.ROOT, "-fx-font-family: \"%s\";-fx-font-size: %.1fpx;", ...);
}
```

This agent must replicate that pattern for the settings module.

## Task 1 — Inject theme token variables into `SettingsPanel` root

**File:** `papiflyfx-docking-settings/src/main/java/org/metalib/papifly/fx/settings/ui/SettingsPanel.java`

### 1a. Add a `paletteFromTheme` helper

Build a `UiCommonPalette` from the `Theme` object using the static helpers already on `UiCommonThemeSupport`:

```java
private static UiCommonPalette paletteFromTheme(Theme theme) {
    Theme resolved = UiCommonThemeSupport.resolvedTheme(theme);
    Color bg = UiCommonThemeSupport.background(resolved);
    Color accent = UiCommonThemeSupport.accent(resolved);
    Color text = UiCommonThemeSupport.textPrimary(resolved);
    Color border = UiCommonThemeSupport.border(resolved);
    boolean dark = UiCommonThemeSupport.isDark(bg);
    return new UiCommonPalette(
        UiCommonThemeSupport.headerBackground(resolved),   // surfaceOverlay
        border,                                             // borderDefault
        text,                                               // textPrimary
        UiCommonThemeSupport.alpha(text, 0.66),             // textMuted
        UiCommonThemeSupport.alpha(text, 0.50),             // textDisabled
        border,                                             // surfaceControl
        UiCommonThemeSupport.alpha(accent, 0.12),           // surfaceControlHover
        UiCommonThemeSupport.alpha(accent, 0.20),           // surfaceControlPressed
        accent,                                             // borderFocus
        accent,                                             // accent
        UiCommonThemeSupport.semanticColor(dark, UiCommonThemeSupport.SemanticTone.SUCCESS),
        UiCommonThemeSupport.semanticColor(dark, UiCommonThemeSupport.SemanticTone.WARNING),
        UiCommonThemeSupport.semanticColor(dark, UiCommonThemeSupport.SemanticTone.DANGER),
        accent,                                             // dropHint
        UiCommonThemeSupport.alpha(Color.BLACK, 0.25)       // shadowOverlay
    );
}
```

### 1b. Add an `applyThemeTokens` method

```java
private void applyThemeTokens(Theme theme) {
    UiCommonPalette palette = paletteFromTheme(theme);
    setStyle(UiCommonThemeSupport.themeVariables(palette) + UiStyleSupport.metricVariables());
}
```

### 1c. Wire the theme listener

Replace the current no-op `themeListener` body:

```java
// BEFORE (no-op):
themeListener = (obs, oldTheme, newTheme) -> {
    // CSS handles theme switching automatically via tokens
};

// AFTER:
themeListener = (obs, oldTheme, newTheme) -> applyThemeTokens(newTheme);
```

Call `applyThemeTokens(runtime.themeProperty().get())` at the end of the constructor (after `loadCategories()`).
