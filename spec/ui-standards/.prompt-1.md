(`code`, `tree`, `github`). A high-level plan exists in `spec/ui-standards/plan.md`.

**Objective**: As the `@ui-ux-designer`, execute **Phase 1: Audit & Standards Definition**.

**Instructions**:
1. **Audit**: Review the CSS files and programmatic UI code in the following modules:
    - `papiflyfx-docking-github` (e.g., `github-toolbar.css`)

2. **Research**: Document specific inconsistencies (hardcoded colors, magic numbers, lack of theme binding) in `spec/ui-standards/research.md`.

4. **Update Plan**: Update the checkboxes in `spec/ui-standards/plan.md` as you complete these sub-tasks.

**Constraints**:
- Follow the **Spec-First** workflow.
- Ensure all proposed changes are compatible with the existing `Theme` record and property binding system.
- Maintain the multiagent workflow (Lead: `@ui-ux-designer`, Reviewers: `@core-architect`, `@spec-steward`).
