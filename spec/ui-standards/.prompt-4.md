# Task: UI Standardization - Phase 4: Validation & Testing

**Context**: Phase 3 (Module-Level Refactoring) is complete. All individual modules have been refactored to use the new shared components and CSS variables.

**Objective**: As the `@qa-engineer` (Lead), validate the consistency and correctness of the UI changes across all modules and themes.

**Instructions**:
1. **Visual & Layout Assertions**: Update existing `FxTest` classes (in `code`, `tree`, `github`, etc.) to include assertions for the new spacing system (4px grid) and shared component states (Hover, Active, Selected).
2. **Theme Switching**: Verify that switching between Light and Dark modes works correctly across all modules, and that no hardcoded colors persist.
3. **Layout Density**: Ensure that layout density settings (Compact vs. Spacious) are correctly applied to the refactored components.
4. **Headless Testing**: Execute the full test suite in headless mode to ensure CI stability.
    - Command: `./mvnw -Dtestfx.headless=true test`
5. **Collaboration & Review**:
    - `@qa-engineer`: Execute the validation plan and update test infrastructure.
    - `@ui-ux-designer`: Act as the reviewer to confirm that visual regressions are addressed.
6. **Update Progress**: Update the checkboxes in `spec/ui-standards/plan.md` (Phase 4).

**Constraints**:
- Follow the **Spec-First** workflow.
- Ensure all UI tests are deterministic and pass in the current environment.
- Maintain the multiagent workflow (Reviewers: `@ui-ux-designer`, `@ops-engineer`).

**Next Step**: Provide a report of any identified UI regressions or test failures.
