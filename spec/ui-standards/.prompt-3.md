# Task: UI Standardization - Phase 3: Module-Level Refactoring

**Context**: Phase 2 (Shared Component Extraction) is complete. Shared UI components (Pills, Chips, Status Slots)
are now available in the shared library, and CSS variables are defined in the `Theme` API.

**Objective**: As the `@feature-dev` (Lead), refactor individual modules to align with the new UI standards and utilize
the shared components.

**Instructions**:
1. **GitHub Module**: Refactor `github-toolbar.css` and related UI code. Replace local implementations of Pills/Chips with the new shared components.
2. **Code Module**: Update `CodeEditorTheme` and canvas rendering mappers to use the new shared color tokens and 4px spacing grid.
3. **Tree Module**: Update `TreeViewTheme` and ensure the canvas-based virtualized tree respects the new visual states (Hover, Active, Selected).
4. **Media & Hugo Modules**: Verify full `Theme` property binding and ensure these modules align with the unified layout density (compact vs. spacious).
5. **Collaboration & Review**:
   - `@feature-dev`: Implement the changes across all modules.
   - `@ui-ux-designer`: Act as the mandatory reviewer to ensure visual fidelity and correct use of the standard tokens.
6. **Update Progress**: Update the checkboxes in `spec/ui-standards/plan.md` (Phase 3).

**Constraints**:
- Follow the **Spec-First** workflow.
- Do NOT introduce any new hardcoded colors or magic numbers.
- Ensure no regressions in existing module functionality.
- Maintain the multiagent workflow (Reviewers: `@ui-ux-designer`, `@qa-engineer`).

**Next Step**: List the specific modules targeted for the first round of refactoring and any identified migration risks.
