# Task: UI Standardization - Phase 2: Shared Component Extraction

**Context**: Phase 1 (Audit and Standards Definition) is complete. We have a set of shared CSS variable tokens and spacing guidelines documented in `spec/ui-standards/research.md`.

**Objective**: As the `@ui-ux-designer` (Lead), with support from `@feature-dev`, identify and extract common UI patterns into a shared location.

**Instructions**:
1. **Identify Components**: Identify specific UI elements used across multiple modules (`Pills`, `Chips`, `StatusSlots`, `Popups`) in the `github`, `code`, and `tree` modules.
2. **Move to Shared Location**: 
   - Move these components to `papiflyfx-docking-api` (if lightweight) or propose the creation of a new `papiflyfx-docking-ui-common` module.
   - Refactor the extracted components to use the new shared CSS variables and spacing system defined in Phase 1.
3. **Collaboration**:
   - `@ui-ux-designer`: Lead the refactoring of CSS and visual properties.
   - `@feature-dev`: Ensure the extracted components maintain their functional logic and proper `Theme` property binding.
4. **Update Progress**: Update the checkboxes in `spec/ui-standards/plan.md` (Phase 2).

**Constraints**:
- Follow the **Spec-First** workflow.
- Ensure all extracted components remain drop-in replacements for their original implementations.
- Maintain the multiagent workflow (Reviewers: `@feature-dev`, `@core-architect`).

**Next Step**: Confirm the list of components identified for extraction and their target shared module.
